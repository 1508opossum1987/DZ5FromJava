Блок 1

🟢 Простая — Коллекции

Создай список студентов.

Используй:

ArrayList<String>

Требования:

добавить минимум 5 студентов;

вывести список;

удалить одного студента по имени;

проверить, есть ли студент "Иван";

вывести количество студентов.



---

🟡 Средняя — Работа с файлами

Создай программу, которая:

1. Создаёт файл notes.txt.


2. Записывает в него несколько строк текста.


3. Затем читает файл.


4. Выводит содержимое в консоль.



Используй BufferedWriter и BufferedReader (или Files.write() и Files.readAllLines()).


---

🔴 Сложная — Stream API

Есть список чисел:

List<Integer> numbers

При помощи Stream API:

оставить только чётные;

умножить каждое число на 2;

отсортировать по убыванию;

вывести результат;

найти сумму получившихся чисел.


Не используй обычные циклы для обработки списка.

Блок 2

🔥 Сложная — Коллекции + Stream API

Создай класс:

Student

Поля:

name

age

grade (средний балл)


Создай список из 10 студентов.

При помощи Stream API:

найти всех старше 18 лет;

отсортировать по среднему баллу;

вывести только имена студентов с баллом выше 4.5;

найти студента с самым высоким баллом;

вычислить средний балл всей группы.

☠️ Суперсложная — Работа с файлами + Коллекции

Есть файл products.txt.

Каждая строка имеет вид:

Хлеб;80
Молоко;120
Сыр;450
...

Необходимо:

считать файл;

создать объект Product для каждой строки;

сохранить все товары в коллекцию;

вывести товары дороже 200;

отсортировать товары по цене;

найти самый дорогой товар;

вычислить общую стоимость всех товаров.


Используй коллекции и Stream API там, где это уместно.


---

💀Убийца*— Всё вместе

Разработай мини-систему учёта библиотеки.

Создай класс:

Book

Поля:

title

author

year

pages

available (выдана книга или нет)


Требования:

1. Считать список книг из файла.


2. Сохранить книги в коллекцию (ArrayList<Book>).


3. Используя Stream API:

вывести книги определённого автора;

найти самую старую книгу;

найти самую новую книгу;

отсортировать книги по названию;

отсортировать по году издания;

вывести только доступные книги;

посчитать среднее количество страниц;

найти книгу с максимальным количеством страниц;

сгруппировать книги по автору.



4. После обработки сохранить обновлённый список обратно в файл.


5. Не использовать циклы там, где задачу можно решить через Stream API.



*Эта задача уже требует уверенного владения коллекциями, Stream API и работой с файлами одновременно и по сложности соответствует хорошему итоговому практическому заданию.
